Rules can be created for when a call have not been touched for that many days or are overdue or close to overdue. You

as a manager can be notified when a Task / Call is overdue or close, according to the rules you place

in this table.

 

 

 

 

Major cases related to escalation time and Frequency parameters

 

Escalation Time    Frequency       Description_________________________

 

0                            0                      Start immediately and once a time

 

0                            1                      Start immediately and repeat after 1 hour

 

1                            0                      Start after/before 1 day and once a time

 

1                            1                      Start after/before 1 day and repeat after 1 hour

 

For Additive/Conditional

 

0                            0                      immediately and once a time

 

……………………                        all cases are same as above defined

 

Special cases of Additive/Conditional

 

 0                           -1                     Ignore Frequency

 

-1                           0                     Ignore Escalation time

 

-1                           -1                     Ignore both

 

 

 

For entering the data in case of Additive or conditional just leave blank Time and Frequency boxes.

System will automatically convert that blank value into -1.

 

 

If you want to escalate how many days in advance you would like to be notice before the Call or Task is

past due. Example 1 day means you are going to be notified 1 day before the Call / Task has it finish date. 

 

0 means that you will be noticed when it should be completed BUT if the Task or Call is closed then the rule

does not apply.
