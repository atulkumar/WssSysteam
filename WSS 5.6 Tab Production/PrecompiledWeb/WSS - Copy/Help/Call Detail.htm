Purpose:

 

All things in WSS start with a CALL. A CALL can be an error, a question, a project task etc. A CALL is the

log of something needed to be executed.

           

 

 

Call Entry is done on this screen. After entering all the information, we can save the call, then we can fill

the corresponding tasks and actions for this call.

 

Customer:                    It has some predefined options in the UDC. Click on the Plus sign and the pop

up window will open. We can select the option from there.

 

Project:                     Project is a mandatory field here. Call Will not be saved if Project is not selected.

                                    It has pre-defined options from the UDC and it displays the name of the various

projects. 

 

Agreement:                It has some predefined options from the UDC. We can select the agreement

no/Type from the UDC.

 

Call Type:                    It has some predefined options in the UDC. Select the one that suits your need.

 

Requested By:           It has some predefined option from the UDC. On clicking the plus sign next to it,

will open a pop up window and we can select the option from there.

Priority:                    It has some predefined options in the UDC like-Normal, Urgent, Minimum,

maximum, high or Low.

 

Requested Date:          By default, it automatically picks up the current date.

 

Est Close Date:            Here, we can specify the finish date of the call. On clicking the calendar icon,

a calendar will open, and then we can select the date from there.

 

Reference:                   To give any reference to your call for quick search

 

Call Entered By:           By default, it takes the current user name who is logged in.

 

 

Template Type:           There are three types of template. They are predefined in the UDC for eg (Call

and task type, Task only and call only.) Select any one the template and it will

open a new pop-up window which will show you the template that has been created

under that template type.

 

Template:                  This field will fill up automatically whatever selection you have made in the template

type field

 

Cat Code 1, Cat Code2, Cat Code 3 It accepts only Numeric values. These fields are optional.

 

            Status:                     When we open a new call, then the status is OPEN, When we assign the call to

         the task owner, then the status is ASSIGNED. When, the task owner enters the

                                             respective tasks and actions on the call, then status becomes PROGRESS. And

                                             finally, when all the task are closed, then status is CLOSED. We can also re-open

                                             the call, then status will become RE-OPEN.

 Comment:                We can add a special comment on the call. Click on the Comment Icon 

 and a pop up window will open. We can type the comment there

and save it. After adding the comment, the color of the Icon changes to Yellow,

indicating that there is a special comment attached here. If the comment is unread,

then in the icon , there will be three-four black lines, indicating that the comment

is still unread and after reading the comment the black lines disappear.

User will be able to add a comment only after saving the new call.

 

 Attachments:           We can attach files as attachments with this call. Click on the Attachments Icon

, a pop up window will open, we can Browse the file and then attach it by

clicking the Upload button.

 

Subject:                    Enter the Subject of the call. (Mandatory)

 

Description:              Give a brief description of the call. (Mandatory)

 

  

 Buttons

 

OK                                     If clicked on OK button this will save your entry and take you to the call view screen

 SAVE                               After, entering the new call, we can click on the SAVE button to save the new call.

 REFRESH                       It will refresh the page, all changes that were not saved will be deleted.

 DELETE                              It will delete the records.

   ATTACHMENT                   With this button, we can attach files with the call.

 CALL FORWARD                  We can forward the task to the other person, using this button.                                                      

 CANCEL                           If we want to cancel the changes made.

 

             HELP                                  Gives the brief description of the current screen.

             LOGOUT                             Used to logout from WSS.

           

A)        How to Create a New Call

Enter the new call data, attach any files if needed, then click on the SAVE button

 to save the new call in the records. The status of the new will be OPEN, once the task

 is assigned, then the status will automatically become ASSIGNED. Further, on entering the Actions,

 the status of the call will become PROGRESS.

 

 

 

 

 

 

 

 

            Once the call is saved, we can fill the Tasks on the same screen in the Task Panel. And after filling the tasks,

we can also fill the Actions in the Action Panel on the same screen.

 C:                                Comment-- Once the task is entered, the icon for the comment appears. On

clicking the Icon, a window appears where we can enter the comment and

save it. Comment can be Internal/External. Internal comment cannot be viewed by

the client but external comment can be viewed by all. User can also send email to

the specific users on comments. For more details check the Help on comment window.

 

A                                  Attachment-To Attach an attachment user has to double click on the task, by doing

so a pop-up window for task edit Screen will appears. Under this screen an icon  is

Displayed, Click on this icon which will further open another screen where user can

Click on the browse button and select the file he wants to upload and then save the

Attachment.

NOTE- You have to save the task again when attachment is uploaded..

F                                              Form-If under this column, an Icon is displayed, then it indicates that this

Task has a form associated with it. Click on the Form Icon  and it will open

a window with a list of all the forms associated with this task.

- This  Icon indicates that the form has no data.

 - this Icon indicates that the form has data.

 

TO:                               Task Order- This field will be used to sort the order of the ask in a call

Subject                      Enter the Subject of the call

 

TType                         Task Type-Select TType from the drop down menu.. It has some predefined

options.

 

Ownr                            Select the Owner from the drop down menu. It displays a list all the users.

 

Dep                             Dependency-Its optional. Select the value from the drop down menu. If any task has a

Dependency value 1 then it means this all other tasks are dependent on this

task.

 

EstClDate                    Estimated Close Date-The close date of the calendar can be selected from the

calendar. Click on the calendar Icon to open the calendar.

 

EHr                              Estimated Hours-enter the estimated hours needed to complete the task.

 

Act                               Action-If this check box is checked, then it becomes mandatory to enter the Actions for

this task. Unless and until, actions are filled for this task, task cannot be closed.

Incase of mandatory, M is displayed, indicating that Actions are to be filled before

closing the task.

 

Pri                               Priority-It indicates the priority of the task. It has some predefined options which

can be selected from the drop down menu.
