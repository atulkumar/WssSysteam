After we have entered the information in the Address Book and created the User Profile,

next task is to assign the Role Security. In this we can enable and disable or hide,

the options that we don’t want the new user to see and change. This is for the security purposes.

 

When we have several user’s, we don’t want all the users to have all the access rights,

so from the Role Security, we can give the selected rights to the users.

 

For Example, Admin has all the rights to View and make changes anywhere.

Where as, we don’t want all the users to have all rights. So we can restrict the rights

depending upon what kind of Role is assigned to the user.

 

From Role, we can select the access level at the Menu, screen and controls level.

 

A)        At Menu Level

 

From here, we can VIEW or HIDE, the screens that we don’t want the user to access. For Example,

if we Hide the Support and Change Menu’s, then under that particular user’s Login, he wont see

these menu’s.So he won’t be able to view data and information under these Menu’s and further,

he wont be able to make any changes.

 

 

 

B)        Screen Level

 

Then we can select a particular Menu, and further we can apply security at the Screen Level.

 

 

 

From here, we can again, View and Hide, the particular screens, that we don’t want the user to

 view and make changes. If we select View, then user will be able to view all the information and

data under that particular screen. He can navigate through the records.

 

If we select Hide, then he won’t be able to view the screen. So he can’t even see the information

and data in that particular screen.

 

Further, we can also hide, Disable and Enable the buttons on these screens. Select a screen,

and then in the Controls we can select the rights that we want to give.

 

 

c)        Control Level

 

 

From here, we can Hide, View, Disable and enable the buttons according to the access level that

we want to provide to the user.  

 

If we select Hide, user won’t be able to see that button. If we select Disabled, the button will

be disabled, i.e. no action will happen on clicking the button.

 

For Example, for Delete button, if we make it View and Disabled then, user will be able to

View the button the screen but if he clicks on it nothing with happens as its disabled.

 

So, this is for the Security purpose, the user will not be able to Delete any records.

 

Buttons Used

 

  OK                      Save the changes and Exit from the current screen.            

  SAVE                 After, entering the new data, click on the SAVE button to save the changes.

   CLOSE                Close the current screen.

  RESET                 It  will Refresh the page and will discard all the unsaved changes.           

  HELP                   Gives the brief description of the current screen.

  LOGOUT              Used to logout from WSS.
