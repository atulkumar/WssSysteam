Purpose:

 

The values of the user defined code indicate the valid set of data that is specific to your business

for a particular field. Each user defined code is either hard-coded or soft-coded. You can customize

fields in your system by setting up user defined codes to meet the needs of your business environment.

User defined codes are stored in tables that relate to a specific system and code type.

 

When you enter a value in a user defined code field the application checks the associated table to

make sure it is a valid value. If no match is found, the program returns an error message “Type Mismatch”.

A)        Adding Or Changing a User defined Code Type

 

A code type categorizes your user defined code values within a product code. When you add user

defined code values, you are adding them to a specific code type; and therefore the code type must

already exist before adding any values to it.

 

On the system Administration Menu

 

1.       Choose a code type to change or click Add to create a new type. Note (The code with Flag=1

      means you

                  cannot change this UDC as this has been defined for a particular company).

2.       To add a new code type, in the last row of the grid, complete the following fields.

a)                  Product Code (can give any number to your UDC)

b)                  UDC Type (Should be max of  max 5 characters)

c)                   Text (Description)

d)                  Company (company that UDC belongs to, or you can leave it blank which

                        means it

                        belongs to all)

e)                  Flag (If flag=0 means you can update this UDC, if flag=1 means you cannot

                        modify it)

3.       You can change or add multiple code types during the same session by pressing Enter.
